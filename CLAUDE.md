# Digital Twin — Josh's Cognitive Architecture

A living AI system that thinks like Josh, not just works for Josh.

## Architecture
- `core/` — Identity graph, value system, decision model
- `engines/` — Memory engine, reasoning engine, action engine, learning loop
- `data/` — Generated knowledge graph, embeddings, decision logs
- `scripts/` — Build scripts, import tools, maintenance

## Stack
- Python 3 for core intelligence
- JSON knowledge graph (portable, inspectable)
- SQLite for decision logs and learning data
- Claude API for reasoning calls
- Embeddings via local model or API

## Principles
- Every component must reflect Josh's actual cognitive patterns (brain map)
- No generic AI behavior — this thinks in parallel bursts, decides in seconds, converts pain to output
- Everything feeds back into the learning loop
- Proactive > reactive
